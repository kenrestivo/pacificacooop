This is a demo for the Flexac system

This is an autocomplete system based on the XMLHTTPRequest object.

How to use it: simply enter at least 2 characters to trigger the autocomplete feature of each field. While flexac is searching the input field will have red borders but you can continue to type to refine your search.

When a search has been completed, a drop down box will appear. You can select the correct entry with the mouse or with the up and down arrows and the enter key. When you select an item, the input field background color will switch to gray to indicate that the field has a valid chosen entry. You can then move to the next field.

Note that this system has been designed to be used (mostly) on local network, as it intrisicly suffers from network latency. The first field does the lookup on a bunch of 13000 planetoid names. And yes, it is slow. The second field completes the name of an italian province. It searches ~100 items only and it is usable (it also cached).

There is also a 250ms artificially a dded delay between a keypress and the system to react. This is almost unnoticeable, but useful. Trust me.

And what does happen when you, finally, click that "Press here" button? You'll be presented with a data dump of the item you inserted. Note the *_id fields. They are set (and even created, if they don't exists) by Flexac.
